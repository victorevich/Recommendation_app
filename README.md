# MoodMatch

Сервис рекомендации эпизодов сериалов на основе анализа эмоционального состояния пользователя. Вместо жанровых фильтров или рейтингов — свободный текст: пользователь описывает своё настроение, а система подбирает эпизоды, которые с ним резонируют.

---

## Как это работает

Пользователь пишет запрос в свободной форме, например: *«чувствую опустошённость и потерю цели, нужно что-то вдохновляющее»*. Текст преобразуется в вектор эмоционального состояния, который сравнивается с векторами эпизодов в базе. Ранжирование учитывает не только близость к запросу, но и историю предпочтений пользователя.

### ML-пайплайн

```
Запрос пользователя
        │
        ▼
  rubert-embedder          ← DeepPavlov/rubert-base-cased-sentence
  (текст → вектор 768d)
        │
        ▼
  ChromaDB (ANN-поиск)     ← top-N кандидатов по косинусному сходству
        │
        ▼
     Ranker                ← взвешенное ранжирование
        │
        ▼
   Результаты
```

### Формула ранжирования

```
score = β·cos(query, episode) + γ·cos(user_profile, episode) + δ·imdb_normalized

где:
  β = 0.6   — вес близости к текущему запросу
  γ = 0.3   — вес близости к профилю пользователя
  δ = 0.1   — вес рейтинга IMDB (нормализован в [0, 1])
```

Если история предпочтений есть — профиль пользователя строится как взвешенная сумма сессионного и глобального центроидов (`α = 0.5`). При первом визите ранжирование работает только по запросу и IMDB.

### Обратная связь

Лайк / дизлайк на эпизод обновляет векторный профиль пользователя: одобренные эпизоды смещают центроид в сторону их embedding, отклонённые — отталкивают. Это позволяет системе адаптироваться без переобучения модели.

## Стек

| Слой | Технология |
|---|---|
| Язык / фреймворк | Python 3.12, FastAPI |
| ML-модель | `DeepPavlov/rubert-base-cased-sentence` (Sentence-Transformers) |
| Векторная БД | ChromaDB (локально, файловое хранилище) |
| Хранение предпочтений | SQLite (`preferences.db`) |
| Фронтенд | HTML, CSS, Vanilla JS |

        └── js/app.js

## Запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

> При первом запуске Sentence-Transformers автоматически скачает модель (~700 МБ).

### 2. Запуск сервера

```bash
uvicorn main:app --reload
```

Открой [http://localhost:8000](http://localhost:8000)

### 3. Инициализация базы (один раз)

Загрузи датасет через API:

```bash
curl -X POST http://localhost:8000/api/init-db \
  -F "file=@data/dataset-mood.csv"
```

Или через Swagger UI: [http://localhost:8000/docs](http://localhost:8000/docs) → `POST /api/init-db`.

После загрузки ChromaDB сохраняется в `data/chromadb/` — повторная инициализация при перезапуске не нужна.

## API

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/` | Фронтенд |
| `POST` | `/api/search` | Семантический поиск эпизодов по тексту настроения |
| `POST` | `/api/feedback` | Лайк / дизлайк, обновление профиля пользователя |
| `POST` | `/api/init-db` | Загрузка CSV-датасета в ChromaDB |
| `GET` | `/api/db-status` | Количество эпизодов в базе |
| `GET` | `/docs` | Swagger UI |

### Пример запроса к `/api/search`

```json
POST /api/search
{
  "query": "тяжёлый разрыв и поиск внутренней силы",
  "session_id": "abc123",
  "top_k": 10
}
```

## Формат датасета

CSV-файл должен содержать следующие столбцы:

| Колонка | Тип | Описание |
|---|---|---|
| `title` | string | Название сериала |
| `episode` | string | Название эпизода |
| `season` | int | Номер сезона |
| `episode_number` | int | Номер эпизода |
| `description` | string | Описание эпизода (используется для embedding) |
| `imdb_rating` | float | Рейтинг IMDB (1–10) |
| `mood_tags` | string | Теги настроения через запятую |
